Rebel CMS

A fork of the discontinued Umbraco 5

Why?

We believe that Umbraco 5 can be rescued. Lots of good code and functionality can be salvaged.

We have the skills to fix it.

Our short term plan:

  • As per Oren's post we are first focusing on the fundamentals.
  • Possibly ripping out DynamicNode.
  • We want to remove "Nice to haves" and end up with something light-weight that works.
  • Performance and simplicity is our first main goal.

Our long term road map:

  • Performance issues solved
  • Start dealing with backlog and issues
  • A new responsive admin User Interface
  • REST API
  • Heroku and AppHarbour support
  • NoSQL core replacement if there is no other option
  • Mono support
  • Continued development and support for "Umbraco 5" supporters

Who are you?

Open Source

Nothing will ever change that. This will be as Open Source as Umbraco is with the same MIT licence. Contributors welcome!
